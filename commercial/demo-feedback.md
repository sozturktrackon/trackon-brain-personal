## Amee Foods (Dina Thomas)
- **Trial result**: Soft rejection — "still evaluating whether this solution is truly relevant for our day-to-day operations"
- **Root cause (assessed)**: Didn't connect document checking to operational pain during self-run trial
- **Lesson**: Trials without guided onboarding are low-conversion. Prospects need to see the failure mode (a real rejection or discrepancy catch) to feel the value.
- **No specific feature requests recorded**

## Gemini Corporation NV (Rahul Sharma)
- **Demo focus requested**: Automated LC export document verification, compliance checks, error detection, workflow integration, security
- **Objection**: Limited interest in broader compliance features (193-country rules, sanctions) — wanted LC verification only
- **Lesson**: Lead with the narrowest, most concrete pain for each prospect. Don't open with the full feature set.

- **Note**: Response should always include both drag-and-drop UI AND email gateway — Wayne's initial response missed the UI and he pushed back

## General Demo Lessons

1. **Always show both access methods**: drag-and-drop UI and email gateway. Forgetting either creates incomplete picture.
2. **ISBP 821 not 745**: The 2023 revision is current. Using 745 signals outdated knowledge.
3. **Lead with the operational cost cascade**, not the bank rejection fee: demurrage, storage, delays = $10K–$50K+ per shipment.
4. **Banking clients**: Be transparent if they're the first; builds trust. Don't pretend scale you don't have.
5. **Sanctions screening specificity**: State that it covers buyers, sellers, banks, AND shipping lines. Vague "sanctions screening" is less compelling.
