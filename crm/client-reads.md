## Pattern: Customer Misattribution of User Error

Seen with both Mana (ETD/ETA) and Agrisar (Adele's multi-currency issue). Customers report system failure; root cause is their own process. Turkish idiom Murat used internally: *"kedi götünü görmüş, yara zannetmiş"* — "the cat saw its own backside and thought it was a wound."

Response pattern that works:
1. Don't defend the system immediately
2. Pull the data (logs, notifications, ticket records)
3. Present neutral findings — let the facts speak
4. Offer a fix for the business logic gap even if the system was correct
